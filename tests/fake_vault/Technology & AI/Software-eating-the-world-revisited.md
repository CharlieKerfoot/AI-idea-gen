# Software Eating the World, Revisited

Marc Andreessen's 2011 thesis that "software is eating the world" has aged remarkably well, but it needs an update. The original argument was that software companies would displace incumbents across every industry. What actually happened is subtler and more interesting: software ate the world, but the world did not become a collection of software companies.

Healthcare runs on Epic. Logistics runs on SAP. Finance runs on Bloomberg terminals. But the organizations that dominate these industries are not software companies in any meaningful cultural sense. They are legacy institutions that have become dependent on software without being transformed by it. The software layer captured enormous value — Epic's founders became billionaires — but it did not restructure the industries it penetrated.

This is the key distinction Andreessen missed. [[Markets-as-information-systems|Markets function as information systems]], and software dramatically improved the information layer. But industries are not just information. They are also regulatory regimes, professional guilds, physical infrastructure, and trust networks. Software can optimize within these constraints, but it cannot easily dissolve them.

The next phase is AI eating software. The question is whether AI will follow the same pattern — capturing value within existing structures without fundamentally reshaping them — or whether it will be different because AI can automate judgment, not just information processing. I suspect the answer depends on [[Incentive-blindness|incentive structures]]. [[The-missionary-vs-mercenary-founder|Missionary founders]] may try to rebuild industries from scratch, but the mercenaries will do what the software companies did: sell tools to incumbents and get rich without changing anything fundamental.
